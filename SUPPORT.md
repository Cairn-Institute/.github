## Support

This document explains how to get help with CAIRN Institute repositories.

## Technical issues

For software defects, feature requests, or technical debt in a specific repository, please open a GitHub issue in that repository using the provided issue templates.

## Security issues

Please do not report security vulnerabilities through public issues. See SECURITY.md for the responsible-disclosure process.

## Organizational or non-technical requests

Non-technical questions, program requests, and general inquiries should go through Linear, Basecamp, or the official CAIRN Institute contact channels rather than GitHub.

## Community questions

For open-source community questions, architecture discussions, and general questions about a public project, use that repository's GitHub Discussions when enabled.
