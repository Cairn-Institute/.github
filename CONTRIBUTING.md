# Contributing to CAIRN Institute Repositories

Thank you for your interest in contributing to a CAIRN Institute project.

## Default contribution workflow

Meaningful changes should be proposed through a pull request. Create a branch from the default branch, make focused changes, add or update tests where applicable, and open a pull request describing the purpose, changes, and testing performed.

## Branch naming

Use the format type/issue-number-short-description, for example feature/TECH-142-google-login or fix/TECH-153-certificate-download. Approved types include feature, fix, hotfix, security, docs, test, refactor, and chore.

## Commit messages

Use the format type(scope): description, for example feat(auth): add Google Workspace login. Approved types include feat, fix, security, docs, test, refactor, chore, build, and ci.

## Code review

All pull requests require human review before merging. Security-sensitive changes such as authentication, authorization, personal data handling, database migrations, and infrastructure changes require additional review.

## Security issues

Please do not report security vulnerabilities through public issues. See SECURITY.md for the responsible-disclosure process.

## Code of conduct

All contributors are expected to follow the project's CODE_OF_CONDUCT.md.
