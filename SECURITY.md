# Security Policy

## Supported versions

Only the current default branch and the most recent published release of each CAIRN Institute repository are supported with security fixes.

## Reporting a vulnerability

Please do not report security vulnerabilities through public GitHub issues or discussions. Instead, report suspected vulnerabilities privately to the CAIRN Institute Technical Owner through GitHub's private vulnerability reporting feature on the affected repository, or through the official CAIRN Institute security contact listed on the organization website.

## What to expect

Reports will be acknowledged, assessed for severity, and tracked to remediation. Reporters should avoid public disclosure until a fix has been coordinated with the Technical Owner.

## Scope

This policy applies to source code and infrastructure configuration maintained in the cairn-institute GitHub organization. It does not cover third-party services, personal experiments, or repositories outside this organization.

## Responsible disclosure

CAIRN Institute appreciates good-faith efforts to responsibly identify and report vulnerabilities and will work with reporters to understand and address issues promptly.
